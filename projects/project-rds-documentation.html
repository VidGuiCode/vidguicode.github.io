LAB 08 - WINDOWS SERVER SERVICES

Overview

This project documents a centralized Remote Desktop Services environment where students can access a shared Windows desktop and applications securely from their own devices using domain credentials.

Scenario

Single-Server RDS

Windows Server 2022
Lab 08
School Project

Centralized RDS environment with AD DS, DNS, and RD Web Access. Students connect from external Windows 11 PCs using domain credentials without domain-joining.

[AD DS] [DNS] [RD WEB] [SESSION-BASED] [LAN]

Core Components

Server: RDS-SRV01 on domain lab8.local with AD DS, DNS Server, and Remote Desktop Services. The RDS deployment is session-based (Quick Start) with RD Web Access. Clients: Physical Windows 11 PCs using bridged networking.

Access & Security

Access is controlled via the RDS_Students Active Directory group. RD Web Access uses a self-signed certificate, and client name resolution is handled with a hosts file entry for the server hostname.

How Users Connect

Users connect via Remote Desktop (mstsc) using domain credentials or through RD Web Access by launching .rdp files for full desktop sessions or RemoteApps.
