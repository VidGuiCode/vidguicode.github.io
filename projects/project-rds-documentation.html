What it actually does

Students open the same managed Windows desktop and apps from their own laptops — nothing to install.

Sign in with your school account

Overview

This project documents a centralized Remote Desktop Services environment where students sign in to a shared Windows desktop and applications securely from their own devices using domain credentials, without joining the domain.

What Runs On The Server

A single Windows Server 2022 box (RDS-SRV01, domain lab8.local) carries every role: the directory, DNS, the session host, and the web access portal.

Active Directory Domain Services

AD DSlab8.localRDS_StudentsDomain accounts
  • Domain controller for lab8.local: holds the user accounts and the RDS_Students security group that decides who is allowed a remote session.

DNS & Name Resolution

DNS ServerA recordshosts file
  • DNS runs on the same server to resolve the domain. External client PCs reach the host through a hosts-file entry that maps the server name to its LAN IP.

Remote Desktop Services

Session-basedQuick StartRD Session HostConnection Broker
  • Single-server session-based deployment (Quick Start): the RD Session Host serves a shared Windows desktop and published RemoteApps, brokered on the same box.

RD Web Access

RD Web AccessIISself-signed TLS.rdp
  • Browser portal over HTTPS: students sign in and launch .rdp files for a full desktop or individual apps. Served with a self-signed certificate for the lab network.

Clients & Networking

Windows 11bridged LANmstscBYOD
  • Physical Windows 11 PCs on a bridged LAN connect with mstsc using domain credentials — without ever joining the domain.
