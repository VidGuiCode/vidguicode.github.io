Per-user permissions gate every share — parents read/write, the child is read-only and can't touch Backup. RAID 5 parity lives in the Areca card, so UnRAID sees one disk; Intel RMM3 manages the box out-of-band even when the OS is down.

What this server does

A shared family drive

Photos, media and documents that everyone in the house can open from any computer.

Everyone gets the right access

Parents can edit, the kid is read-only, and the backup folder is locked down — set per person.

Survives a dead disk

Drives are mirrored and parity-protected, so if one disk fails, nothing is lost.

Acts like a local disk too

It can also hand a Windows PC raw storage over the network that shows up like a built-in drive (iSCSI).

Overview

For the Cloud Infrastructure 2 module of my BTS Cloud Computing, my team of two was handed a bare Intel 2U rack server and told to turn it into a working storage box. We fitted the hardware, set up a hardware RAID controller, installed UnRAID, racked it, and built a full family file-sharing setup on top. To finish, we tested iSCSI block storage from a Windows machine.

The Intel SR2600 opened up during staging, showing the dual-CPU mainboard, the DDR3 modules and the front drive bays.

Inside the Server

The Hardware

Intel SR2600 2UDual Xeon X565016 GB DDR3 ECCRedundant PSU
  • The machine we got is an Intel SR2600 2U rack server on an S5520UR board, with dual Intel Xeon X5650 (12 cores total) and 16 GB of DDR3 ECC across six channels.

Storage & RAID

Areca ARC-1880RAID 5 / XFSBtrfs RAID 1SAS + SATA
  • Three enterprise 15K SAS disks that a dedicated RAID controller presents as one logical volume. All the redundancy happens in hardware, so UnRAID just sees a single disk.
  • Two native SATA disks mirrored as a Btrfs RAID 1 pool. Each disk holds a full copy, so losing one drive loses no data. This is where the Backup share lives.

The Software: UnRAID

UnRAIDLinuxDockerCommunity Apps
  • A Linux-based NAS OS that is good at mixing disks of different sizes in one parity-protected array, with a friendly web UI, Docker, VMs and a plugin store. We also dug into how DAS, NAS and SAN differ and the trade-offs of UnRAID's paid licensing.

File Sharing & Permissions

SMB4 sharesper-user ACLWin / Linux / macOS
  • The setup was a family with three accounts and four SMB shares. Photos, Media and Documents live on the big RAID 5 array, and Backup sits on the mirrored RAID 1 pool. The two parent accounts get full read/write, while the child account is read-only on the libraries and can't touch Backup at all.

iSCSI Block Storage

iSCSI targetfile-based LUNIQN ACLNTFS
  • On top of file shares, we handed out block storage over iSCSI. We added a Community Apps plugin, created a file-based LUN on the hardware RAID 5 array, and published it as a target with an ACL locked to one approved Windows initiator (by IQN). On Windows the LUN turns up in Disk Management like a brand-new local disk.

Rack & Remote Management

24U rackIntel RMM3IPMI 2.0out-of-band
  • Once it was working on the bench, the server went onto rails in a 24U rack with tidy cabling. Remote management runs over Intel RMM3 (IPMI 2.0), which gives remote KVM, power control and BIOS access even when the OS is down, next to the UnRAID web interface for everyday admin.

The Areca ARC-1880 on its PCIe riser. This card is what builds and owns the RAID 5 array in hardware.

The iSCSI LUN showing up on Windows as a fresh, unallocated local disk, ready to initialise and format.

Problems Worth Remembering

When the web UI died

After a config reset, nginx threw a 502 and PHP came back blank. Restarting nginx, php-fpm and emhttp by hand fixed it — and adding the daemon to UnRAID's startup go file made it stick.

Reviving the Btrfs pool

The mirror refused to start with "Too many wrong and/or missing disks." A fresh disk assignment (New Config, keeping the assignments) and reformatting the unmounted disks brought it back.
